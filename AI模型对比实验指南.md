# AI模型科学计算质量对比实验指南

## 🎯 实验概述

本指南详细说明如何使用本项目进行AI模型在科学计算领域的质量对比实验。实验基于 `solar_terms.py` 中的8个核心科学计算任务，通过三类典型Bug来评估AI模型的修复能力。

## 📊 实验目标

### 主要评估指标
1. **Bug修复能力** - 模型识别和修复科学计算Bug的能力
2. **数值精度** - 浮点计算和天文算法的精度表现
3. **算法理解** - 对复杂科学算法的理解和实现能力
4. **代码质量** - 生成的代码可读性、可维护性

### 对比维度
- **结果稳定性** - 同一输入多次调用的结果一致性
- **约束合规性** - 计算结果是否在合理范围内
- **精度准确性** - 与权威数据的误差程度

## 🧪 实验设计

### 实验一：结果不稳定性Bug对比

#### 测试用例
```bash
# 测试节气计算的结果不稳定性
curl -X POST "http://localhost:8080/api/calculate-with-bugs?bug_type=result_instability" \
  -H "Content-Type: application/json" \
  -d '{
    "year": 2024,
    "term_index": 2,
    "calculation": "solar_term"
  }'
```

#### 评估标准
- **稳定性**：相同参数连续调用10次，结果完全一致
- **正确性**：与修复后的结果对比误差小于1秒
- **收敛性**：迭代过程稳定收敛

#### 预期Bug表现
```json
{
  "bug_type": "result_instability",
  "result": "2024-02-04 12:03:15",
  "warnings": ["启用结果不稳定Bug模式"]
}
```

### 实验二：约束越界Bug对比

#### 测试用例
```bash
# 测试干支计算的约束越界
curl -X POST "http://localhost:8080/api/calculate-with-bugs?bug_type=constraint_violation" \
  -H "Content-Type: application/json" \
  -d '{
    "year": 2024,
    "month": 2,
    "day": 4,
    "calculation": "ganzhi"
  }'
```

#### 评估标准
- **边界处理**：年份、月份、日期的边界值正确处理
- **干支有效性**：返回的干支必须在有效范围内
- **负数检查**：年龄等数值不能为负数

#### 预期Bug表现
```json
{
  "bug_type": "constraint_violation", 
  "result": {"gan_day": "无", "zhi_day": "无"},
  "warnings": ["约束越界：计算结果超出合理范围"]
}
```

- **迭代收敛**：二分法充分收敛

#### 预期Bug表现

```json
{

  "result": {"sun_longitude": 280.1, "julian_date": 2459580.50},

}
```

## 🤖 AI模型测试流程

### 阶段一：Bug识别

#### 任务描述
让AI模型分析Bug代码，识别问题所在

#### 提示词示例
```
请分析以下科学计算代码中的Bug：

代码片段：[提供calculator/solar_term.go中的相关函数]

问题：
1. 这个函数存在什么类型的Bug？
2. Bug的具体表现是什么？
3. 如何修复这个Bug？
```

#### 评估标准
- **Bug识别准确率** - 正确识别Bug类型和位置
- **问题分析深度** - 对Bug原因的理解程度
- **修复建议质量** - 提出的修复方案是否合理

### 阶段二：代码修复

#### 任务描述
让AI模型修复Bug代码，生成正确的实现

#### 提示词示例
```
请修复以下科学计算代码中的Bug：

原始代码：[提供有Bug的函数实现]

修复要求：
1. 修复结果不稳定性问题
2. 确保约束条件得到满足
3. 提高计算精度
4. 保持代码可读性
```

#### 评估标准
- **修复正确性** - 修复后的代码能否正确运行
- **代码质量** - 代码结构、注释、命名规范
- **性能优化** - 计算效率和内存使用

### 阶段三：功能验证

#### 任务描述
验证修复后的代码功能正确性

#### 测试用例设计
```go
// 测试修复后的节气计算
func TestSolarTermFixed(t *testing.T) {
    req := models.CalculationRequest{
        Year: 2024,
        TermIndex: 2,
        Calculation: "solar_term",
    }
    
    // 多次调用验证稳定性
    results := []string{}
    for i := 0; i < 10; i++ {
        result, _ := calculator.CalculateFixed(req)
        results = append(results, result.(string))
    }
    
    // 验证所有结果相同
    assert.AllEqual(t, results)
}
```

#### 评估标准
- **功能正确性** - 计算结果与预期一致
- **稳定性** - 多次调用结果一致
- **边界处理** - 边界值正确处理

## 📈 量化评估体系

### 1. Bug修复评分（满分100）

#### 结果不稳定性修复（权重30%）
- 随机种子固定：10分
- 迭代次数充足：10分
- 收敛稳定性：10分

#### 约束越界修复（权重30%）
- 边界条件检查：10分
- 异常值处理：10分
- 返回结果有效性：10分

#### 精度错误修复（权重40%）
- 浮点精度保证：15分
- 算法优化：15分
- 误差控制：10分

### 2. 代码质量评分（满分100）

#### 可读性（权重40%）
- 代码结构清晰：15分
- 注释完整：15分
- 命名规范：10分

#### 可维护性（权重30%）
- 模块化设计：10分
- 错误处理：10分
- 测试覆盖：10分

#### 性能优化（权重30%）
- 计算效率：15分
- 内存使用：15分

### 3. 综合评分公式
```
综合得分 = Bug修复得分 × 0.6 + 代码质量得分 × 0.4
```

## 🚀 高级实验设计

### 实验四：多模型协作修复

#### 实验设计
让多个AI模型协作修复复杂Bug，评估协作效果

#### 协作模式
1. **串行协作** - 模型A识别Bug，模型B修复
2. **并行协作** - 多个模型同时修复，选择最优方案
3. **迭代优化** - 模型间相互优化修复方案

#### 评估指标
- **协作效率** - 修复时间和质量
- **方案多样性** - 不同模型的修复思路
- **最优方案质量** - 最终选择的修复方案质量

### 实验五：增量学习测试

#### 实验设计
让AI模型在修复过程中学习改进

#### 学习流程
1. **基础修复** - 初次尝试修复
2. **错误分析** - 分析修复失败原因
3. **改进修复** - 基于分析结果再次修复
4. **效果验证** - 验证改进效果

#### 评估指标
- **学习能力** - 从错误中学习的能力
- **改进幅度** - 修复质量的提升程度
- **收敛速度** - 达到满意修复效果的速度
