# 【Doubao-Seed-Code-Dogfood-2.1.1】模型的 BUGFIX: 符号微分计算修复报告

## 问题概述

符号微分计算模块存在核心功能缺陷，对于求导请求无法正确计算导数结果。

### 问题表现

请求示例：
```json
{
  "calculation": "symbolic_calc",
  "params": {
    "operation": "differentiate",
    "expression": "x^3 + 2*x^2 + x",
    "variable": "x"
  }
}
```

异常响应字段：

| 字段 | 异常值 | 预期值 |
|------|--------|--------|
| `derivative` | `"d/dx(x^3 + 2*x^2 + x)"`（占位符） | `"3*x^2 + 4*x + 1"` |
| `simplified` | `""`（空字符串） | `"3*x^2 + 4*x + 1"` |
| `numeric_value` | `0`（恒为0） | 代入变量后的数值结果（如x=1时为8） |
| `parsed_tree` | `null` | 表达式树结构 |

---

## 根本原因分析

### 1. 求导算法未递归实现

**问题代码位置**：`symbolic_calc.go:301-334`

原 `calculateDerivative` 函数仅实现了基础求导规则：
- 常数导数（如 `"5"` → `"0"`）
- 单变量导数（如 `"x"` → `"1"`）
- 简单幂函数（如 `"x^3"` → `"3*x^2"`）

**缺陷**：未实现复合表达式的递归处理，无法对加法、乘法等组合运算（如 `x^3 + 2*x^2`）进行正确求导。

### 2. 表达式树解析缺失

**问题代码位置**：`symbolic_calc.go:243-255`

原 `buildParseTree` 函数仅做了表面分词和运算符查找，未构建真正的抽象语法树（AST）。

**缺陷**：缺少词法分析→语法分析→构建AST的完整流程，无法支持复杂表达式处理。

### 3. 符号化简模块功能不足

**问题代码位置**：`symbolic_calc.go:337-366`

原 `simplify` 函数仅处理了 `0*`、`1*`、`^1`、`^0` 等简单情况。

**缺陷**：
- 未对导数结果进行自动化简
- 缺少同类项合并、常数运算合并等关键化简逻辑

### 4. 数值求值逻辑简化过度

**问题代码位置**：`symbolic_calc.go:369-408`

原 `evaluate` 函数仅支持两个操作数的简单运算。

**缺陷**：
- 无法处理复合表达式求值
- 未正确代入变量值进行计算
- 默认返回0导致结果恒为0

### 5. 错误静默处理机制

**缺陷**：系统在核心计算逻辑失败时仍返回 `success: true`，缺少计算结果有效性检查环节。

## 修复方案

### 核心算法实现要点

#### 1. 表达式解析（Shunting-yard算法）

**实现位置**：`symbolic_calc_fixed.go:242-336`

**算法步骤**：

1. **词法分析（Tokenize）**：
   - 识别数字、变量、运算符、函数
   - 处理负号（如 `-x` → `-1*x`）
   - 维护运算符优先级（^ > * / > + -）

2. **调度场算法（Shunting-yard）**：
```
伪代码：
初始化输出队列和运算符栈
对每个token：
    如果是数字或变量，加入输出队列
    如果是函数，压入运算符栈
    如果是函数参数分隔符（如逗号）：
        弹出运算符栈到输出队列直到遇到左括号
    如果是运算符o1：
        while 运算符栈不为空且栈顶不是左括号：
            o2 = 栈顶运算符
            如果 o2优先级 > o1优先级 或
               (o2优先级 == o1优先级且o1是左结合)：
                弹出o2到输出队列
            else：
                break
        将o1压入栈
    如果是左括号，压入栈
    如果是右括号：
        弹出运算符到输出队列直到遇到左括号
        弹出左括号（不加入输出）
        如果栈顶是函数，弹出到输出队列
将运算符栈中剩余元素弹出到输出队列
```

#### 2. 递归求导规则

**实现位置**：`symbolic_calc_fixed.go:338-504`

**核心求导规则**：

| 规则类型 | 公式 | 实现说明 |
|----------|------|----------|
| 常数 | d/dx(c) = 0 | 返回数字节点 "0" |
| 变量 | d/dx(x) = 1<br>d/dx(y) = 0 | 匹配变量名返回结果 |
| 和 | d/dx(f+g) = f' + g' | 递归求导后相加 |
| 差 | d/dx(f-g) = f' - g' | 递归求导后相减 |
| 积 | d/dx(fg) = f'g + fg' | 乘积法则实现 |
| 商 | d/dx(f/g) = (f'g - fg')/g² | 商法则实现 |
| 幂（常数指数） | d/dx(xⁿ) = nxⁿ⁻¹ | 检查指数为常数时优化 |
| 一般幂 | d/dx(uᵛ) = uᵛ(v'lnu + vu'/u) | 链式法则通用形式 |
| 函数 | d/dx(f(g(x))) = f'(g(x))g'(x) | 先求外层函数导，乘以内层导 |

#### 3. 表达式化简策略

**实现位置**：`symbolic_calc_fixed.go:506-678`

**化简层次**：

1. **常数折叠**：
   - 纯数字运算直接计算（如 `2+3` → `5`）
   - 支持 +、-、*、/、^ 运算

2. **恒等化简**：
   ```
   x + 0 → x
   x * 0 → 0
   x * 1 → x
   x ^ 0 → 1
   x ^ 1 → x
   ```

3. **同类项合并**：
   - 收集所有加法/减法项
   - 按变量部分分组
   - 合并系数（如 `2x + 3x` → `5x`）

4. **递归化简**：
   - 先化简子节点再化简当前节点
   - 二次化简确保结果最简

#### 4. 数值求值保证

**实现位置**：`symbolic_calc_fixed.go:680-764`

**求值流程**：

1. 递归遍历AST节点
2. 变量节点替换为变量值
3. 运算符节点递归计算左右子树后执行运算
4. 函数节点计算参数后代入函数值

**支持的函数**：sin, cos, tan, ln, log, sqrt, exp

## 新增接口说明

### 1. 修复版计算接口

**端点**：`POST /api/calculate-fixed`

**请求格式**（与原接口兼容）：
```json
{
  "calculation": "symbolic_calc",
  "params": {
    "operation": "differentiate",
    "expression": "x^3 + 2*x^2 + x",
    "variable": "x",
    "x_value": 1.0
  }
}
```

**响应格式**：
```json
{
  "success": true,
  "result": {
    "original_expression": "x^3 + 2*x^2 + x",
    "result_expression": "3*x^2+4*x+1",
    "parsed_tree": { /* AST结构 */ },
    "derivative": "3*x^2+4*x+1",
    "simplified": "3*x^2+4*x+1",
    "numeric_value": 8.0,
    "variables": { "x": 1.0 },
    "operation_type": "differentiate"
  },
  "timestamp": "...",
  "calculation": "symbolic_calc_fixed",
  "session_id": "..."
}
```

### 2. 对比测试接口

**端点**：`POST /api/solver/compare`

**响应格式**：
```json
{
  "original": {
    "result": { /* 原始计算结果 */ },
    "error": ""
  },
  "fixed": {
    "result": { /* 修复后计算结果 */ },
    "error": ""
  },
  "diff": {
    "fields": {
      "derivative": {
        "original": "d/dx(...)",
        "fixed": "3*x^2+4*x+1",
        "changed": true
      }
    }
  },
  "is_fixed": true
}
```

## 测试用例验证

### 测试用例1：多项式求导

**输入**：`x^3 + 2*x^2 + x`

**预期导数**：`3*x^2 + 4*x + 1`

**x=1时数值**：`3(1)^2 + 4(1) + 1 = 8`

**验证点**：
- derivative字段不再是占位符
- simplified字段有化简结果
- numeric_value=8（非0）

### 测试用例2：三角函数求导

**输入**：`sin(x) + cos(x)`

**预期导数**：`cos(x) - sin(x)`

**x=0时数值**：`cos(0) - sin(0) = 1 - 0 = 1`

### 测试用例3：乘积法则

**输入**：`x^2 * sin(x)`

**预期导数**：`2x*sin(x) + x^2*cos(x)`
